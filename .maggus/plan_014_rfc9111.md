# Plan: RFC 9111 (HTTP Caching) — 99% Client Compliance

## Introduction

RFC 9111 has 96 client-side MUSTs. After plan_011 (TASK-014-021), 95 are covered (99%). This plan documents full coverage and the 2 additional audit tasks.

## Goals

- RFC 9111 client compliance from 90.6% to 99% (95/96)
- Remaining 1 item: disconnected stale serving (§4.2.4) — deferred by design (MAY, not MUST)

## Preconditions

- **Age header generation**: `CacheBidiStage` injects `Age` header when serving from cache
- **Qualified no-cache/private parsing**: `CacheControl.NoCacheFields` and `CacheControl.PrivateFields` properties exist in `src/TurboHttp/Protocol/RFC9111/CacheControl.cs`
- **must-understand directive**: `CacheControl.MustUnderstand` property exists
- **Incomplete response guard**: `HttpCacheStore.Put()` rejects 206 and Content-Range responses
- **Cache invalidation**: `CacheBidiStage` invalidates Location/Content-Location URIs on unsafe method + 2xx
- **HEAD freshening**: `CacheValidationRequestBuilder.BuildHeadValidation()` exists

## User Stories

### TASK-001: Trailer/Header Separation Audit

**Description:** As a developer, I want to verify that cached responses do not merge trailer fields into the header block.

**RFC**: 9111 §3.1 — "MUST NOT combine trailer fields with header fields"

**Acceptance Criteria:**
- [ ] Audit: CacheEntry.Response.TrailingHeaders separate from Headers
- [ ] Test: Cached chunked response with trailers → trailers not in headers
- [ ] No code change needed if already correct

**Tests** (extends `src/TurboHttp.Tests/RFC9111/04_CacheStoreTests.cs`):

| DisplayName | Method |
|-------------|--------|
| `RFC9111-3.1-CS-036: Trailers not merged into cached headers` | `Should_NotMergeTrailers_When_CachedWithTrailers` |

**Effort**: 1h

### TASK-002: Connection-Specific Header Filtering

**Description:** As a developer, I want connection-specific headers (Connection, Keep-Alive, Proxy-Authenticate) excluded from cache storage.

**RFC**: 9111 §3.1 — "MUST exclude connection-specific headers"

**Implementation**:
- File: `src/TurboHttp/Protocol/RFC9111/HttpCacheStore.cs`
- Change: In `Put()` before storage — remove connection-specific headers
- List: `Connection`, `Keep-Alive`, `Proxy-Authenticate`, `Proxy-Authorization`, `TE`, `Trailer`, `Transfer-Encoding`, `Upgrade`

**Acceptance Criteria:**
- [ ] Connection-specific headers removed before cache storage
- [ ] Other headers preserved
- [ ] Unit tests written and passing

**Tests** (extends `src/TurboHttp.Tests/RFC9111/04_CacheStoreTests.cs`):

| DisplayName | Method |
|-------------|--------|
| `RFC9111-3.1-CS-037: Connection header not stored in cache` | `Should_NotStore_When_ConnectionHeader` |
| `RFC9111-3.1-CS-038: Keep-Alive header not stored in cache` | `Should_NotStore_When_KeepAliveHeader` |
| `RFC9111-3.1-CS-039: Custom headers preserved in cache` | `Should_Store_When_CustomHeaders` |

**Effort**: 1.5h
