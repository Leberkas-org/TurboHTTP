## Coverage Details

## Implementation Notes

- **Persistent connections:** `ConnectionReuseEvaluator` (RFC 9112 §9) determines keep-alive vs. close based on response headers and HTTP version.
- **Chunked transfer:** Full encoding and decoding of chunked bodies including chunk extensions and trailers.
- **Pipelining:** `CorrelationHttp1XStage` maintains FIFO request-response matching.
- **Per-host limits:** `PerHostConnectionLimiter` enforces configurable concurrency caps.
- **Host header:** Always injected by the encoder; connections are keyed by `HostKey` (scheme + host + port + version).
