## Implementation Notes

- **Static table:** 61 pre-defined header name/value pairs (Appendix A); implemented as a static lookup.
- **Dynamic table:** FIFO eviction with 32-byte per-entry overhead (`HpackDynamicTable`). Encoder and decoder maintain synchronised tables per connection.
- **Huffman coding:** `HuffmanCodec` implements the RFC 7541 Huffman code table for both encoding and decoding.
- **Sensitive headers:** `Authorization`, `Cookie`, and similar headers automatically use `NeverIndex` representation (§7.1.3) to prevent compression-based attacks.
- **Table size updates:** `SETTINGS_HEADER_TABLE_SIZE` changes trigger a dynamic table resize signal in the next header block.
- **Error handling:** `HpackException` is thrown on RFC 7541 violations (e.g. invalid index, table size mismatch).
