## Implementation Details

### CookieJar

The `CookieJar` is the central cookie store:

- **Thread-safe:** All operations are safe for concurrent use across multiple connections.
- **`Set-Cookie` parsing:** Processes `Set-Cookie` response headers including all attributes.
- **`Cookie` injection:** Adds matching cookies to outbound request `Cookie` headers.

### Cookie Matching (§5.4)

Cookie selection for a request follows the RFC 6265 matching algorithm:

| Attribute | Matching Rule |
|-----------|--------------|
| Domain | Domain suffix match (`.example.com` matches `sub.example.com`) |
| Path | Path prefix match (`/api` matches `/api/v1/users`) |
| Secure | Only sent to `https://` origins |
| HttpOnly | Not accessible from JavaScript; sent on all HTTP requests |

### Cookie Attributes (§5.2)

| Attribute | Behaviour |
|-----------|-----------|
| `Max-Age` | Cookie lifetime in seconds; takes precedence over `Expires` |
| `Expires` | Absolute expiry date (HTTP-date format) |
| `Secure` | Cookie only sent over TLS connections |
| `HttpOnly` | Cookie not exposed to client-side scripts |
| `SameSite` | Cross-site request policy (`Strict`, `Lax`, `None`) |
| `Domain` | Sets the domain scope for the cookie |
| `Path` | Sets the path scope for the cookie |

Session cookies (no `Max-Age` or `Expires`) are held in memory only and discarded when the client is disposed.
