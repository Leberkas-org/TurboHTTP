## Implementation Details

### RedirectHandler (§15.4)

`RedirectHandler` processes 3xx responses according to RFC 9110:

| Status | Method Rewrite | Condition |
|--------|---------------|-----------|
| 301 Moved Permanently | GET (non-GET requests) | Safe rewrite |
| 302 Found | GET | Safe rewrite |
| 303 See Other | Always GET | Unconditional |
| 307 Temporary Redirect | Preserved | Idempotent methods only |
| 308 Permanent Redirect | Preserved | Idempotent methods only |

- **HTTPS→HTTP protection:** Redirects from `https://` to `http://` are blocked by default.
- **Loop detection:** Tracks visited URLs to detect and abort redirect loops.
- **Configurable policy:** `RedirectPolicy` in `TurboClientOptions` controls max redirects and security checks.

> **Note:** RFC9110-15.4-RH-015 (relative Location URI resolution) is a known pre-existing test failure.

### RetryEvaluator (§9.2)

`RetryEvaluator` determines whether a failed request is safe to retry based on idempotency:

- **Safe/idempotent methods:** GET, HEAD, PUT, DELETE, OPTIONS, TRACE — always retryable on connection failure.
- **Non-idempotent methods:** POST, PATCH — only retried if the server explicitly sends `Retry-After`.
- **Retry-After parsing:** Supports both delta-seconds and HTTP-date formats.
